What does this app do?

This app connects your Homey to the hail warning system hagelschutz-einfach-automatisch.ch operated by the VKF (Association of Cantonal Fire Insurers) and SRF Meteo. It polls the METEO REST API every 120 seconds and automatically triggers Homey Flows as soon as a hail warning becomes active for your location.

ℹ️ This service is available in Switzerland only and requires prior registration at hagelschutz-einfach-automatisch.ch.

Features

🌨️ Automatic hail warning

As soon as a hail warning becomes active, the app triggers a Flow – e.g. to raise all blinds immediately.

✅ All-clear detection

When the warning is lifted, a separate Flow is triggered so blinds can be lowered again.

🔌 API connection monitoring

If an API request fails, an alarm is triggered immediately and a Flow is fired.

⏱️ Watchdog

If no successful API poll has occurred for more than 10 minutes, a Flow is triggered automatically.

Capabilities / Sensors

The «Hail Warning Sensor» device provides the following values on the device tile:

🚨 Hail Warning alarm_generic

Boolean alarm – true when currentState ≠ 0. Displayed by Homey as a red alarm indicator.

# Hail Signal hail_state

Raw numeric value from the API:

  • 0 – No hail
  • 1 – Hail warning active
  • 2 – Hail warning active (test alarm)
🔌 API Connection Error api_error_state

Boolean alarm – true when the last API request failed (no HTTP 200 or timeout).

🕐 Last API Poll last_poll

Date and time of the last successful API request in DD.MM, HH:MM format.

Test Alarm

The API connection can be tested directly at meteo.netitservices.com. After logging in, activate the Test alarm switch – within 120 seconds the sensor should report currentState = 2 and trigger the Flow. Deactivate the test alarm afterwards.

Alternatively, use the Flow action «Check hail status now» to trigger an immediate manual request.

Frequently Asked Questions

Where do I get the Device ID and hwtypeId?

After registering at hagelschutz-einfach-automatisch.ch you receive the credentials by email. The Device ID is the 12-character MAC address of your Signalbox; the hwtypeId is an integer device type identifier.

Why is the poll interval fixed at 120 seconds?

The VKF API specification requires a minimum interval of 120 seconds. The hail warning system recalculates the forecast only every 5 minutes anyway – more frequent requests provide no benefit.

What does currentState = 2 mean?

currentState = 2 is a test alarm that can be triggered manually via meteo.netitservices.com. The app treats it identically to a real alarm (non-zero = hail).

The sensor shows «unavailable» – what to do?

  • Check the Homey internet connection
  • Verify Device ID and hwtypeId in the device settings
  • Log in to meteo.netitservices.com and check the status
  • Run the Flow action «Check hail status now» and check the logs

Does the app work without a Signalbox?

Yes – the app uses the METEO REST API directly and does not require a physical Signalbox (VKF box). It replaces it for systems with their own building automation.
