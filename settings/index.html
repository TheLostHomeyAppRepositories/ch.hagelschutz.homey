Was macht diese App?

Die App verbindet Ihren Homey mit dem Hagelwarnsystem hagelschutz-einfach-automatisch.ch der Vereinigung Kantonaler Feuerversicherungen (VKF) und SRF Meteo. Sie fragt alle 120 Sekunden die METEO REST API ab und löst automatisch Homey-Flows aus, sobald eine Hagelwarnung für Ihren Standort aktiv wird.

ℹ️ Der Dienst ist ausschliesslich in der Schweiz verfügbar und erfordert eine vorgängige Registrierung auf hagelschutz-einfach-automatisch.ch.

Funktionen

🌨️ Automatische Hagelwarnung

Sobald eine Hagelwarnung aktiv wird, löst die App einen Flow aus – z.B. zum Hochfahren aller Storen.

✅ Entwarnung

Bei Entwarnung wird ein separater Flow ausgelöst, damit die Storen wieder geschlossen werden können.

🔌 API-Verbindungsüberwachung

Schlägt eine API-Abfrage fehl, wird sofort ein Alarm ausgelöst und ein Flow getriggert.

⏱️ Watchdog

Wenn seit mehr als 10 Minuten keine erfolgreiche API-Abfrage stattgefunden hat, wird automatisch ein Flow ausgelöst.

  • Registrierung auf hagelschutz-einfach-automatisch.ch
  • Gültige Geräte-ID (deviceId / Seriennummer) und Hardware-Typ-ID (hwtypeId)

Capabilities / Sensoren

Das Gerät «Hagelwarn-Sensor» stellt folgende Werte auf dem Geräte-Tile bereit:

🚨 Hagelwarnung alarm_generic

Boolescher Alarm – true wenn currentState ≠ 0. Wird von Homey als roter Alarm-Indikator dargestellt.

# Hagelsignal hail_state

Numerischer Rohwert der API:

  • 0 – Kein Hagel
  • 1 – Hagelwarnung aktiv
  • 2 – Hagelwarnung aktiv (Testalarm)
🔌 API-Verbindungsfehler api_error_state

Boolescher Alarm – true wenn die letzte API-Abfrage fehlgeschlagen ist (kein HTTP 200 oder Timeout).

🕐 Letzte API-Abfrage last_poll

Datum und Uhrzeit der letzten erfolgreichen API-Abfrage im Format TT.MM, HH:MM.

Geräte-Einstellungen

Die folgenden Einstellungen können nach dem Hinzufügen des Geräts unter Geräte → Hagelwarn-Sensor → Einstellungen angepasst werden.

Einstellung Beschreibung Standard
Geräte-ID
device_id		 12-stellige Seriennummer / MAC-Adresse der Signalbox. Erhalten nach Registrierung. –
Hardware-Typ-ID
hwtype_id		 Ganzzahliger Gerätetyp-Identifikator aus der Registrierungsbestätigung. 1
Abfrageintervall
poll_interval		 Wie oft die API abgefragt wird (in Sekunden). Minimum: 120 s gemäss API-Spezifikation. 120 s
⚠️ Das Abfrageintervall darf gemäss API-Spezifikation der VKF nicht kürzer als 120 Sekunden eingestellt werden.

Testalarm

Die API-Verbindung lässt sich direkt auf meteo.netitservices.com testen. Nach dem Login den Testalarm-Schalter aktivieren – innerhalb von 120 Sekunden sollte der Sensor currentState = 2 melden und der Flow auslösen. Den Testalarm danach wieder deaktivieren.

Alternativ kann die Flow-Aktion «Hagelstatus jetzt prüfen» genutzt werden, um sofort eine manuelle Abfrage auszulösen.

Häufige Fragen

Wo bekomme ich die Geräte-ID und hwtypeId?

Nach der Registrierung auf hagelschutz-einfach-automatisch.ch erhalten Sie die Zugangsdaten per E-Mail. Die Geräte-ID ist die 12-stellige MAC-Adresse Ihrer Signalbox, die hwtypeId ein ganzzahliger Identifikator des Gerätetyps.

Warum ist das Abfrageintervall auf 120 Sekunden fixiert?

Die API-Spezifikation der VKF schreibt ein Mindestintervall von 120 Sekunden vor. Das Hagelwarnsystem berechnet die Prognose ohnehin nur alle 5 Minuten neu – häufigere Abfragen bringen keinen Mehrwert.

Was bedeutet currentState = 2?

currentState = 2 ist ein Testalarm, der manuell über meteo.netitservices.com ausgelöst werden kann. Die App behandelt ihn identisch wie einen echten Alarm (non-zero = Hagel).

Der Sensor zeigt «nicht verfügbar» an – was tun?

  • Internetverbindung des Homey prüfen
  • Geräte-ID und hwtypeId in den Einstellungen kontrollieren
  • Auf meteo.netitservices.com einloggen und Status prüfen
  • Flow-Aktion «Hagelstatus jetzt prüfen» ausführen und Log beobachten

Funktioniert die App ohne Signalbox?

Ja – die App nutzt die METEO REST API direkt und benötigt keine physische Signalbox (VKF-Box). Sie ersetzt diese für Systeme mit eigener Gebäudesteuerung.
