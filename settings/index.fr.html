À quoi sert cette app ?

Cette app connecte votre Homey au système d'alerte grêle hagelschutz-einfach-automatisch.ch de l'Association des établissements cantonaux d'assurance incendie (AEAI) et SRF Meteo. Elle interroge l'API METEO REST toutes les 120 secondes et déclenche automatiquement des Flows Homey dès qu'une alerte grêle devient active pour votre emplacement.

ℹ️ Ce service est disponible uniquement en Suisse et nécessite une inscription préalable sur hagelschutz-einfach-automatisch.ch.

Fonctionnalités

🌨️ Alerte grêle automatique

Dès qu'une alerte grêle devient active, l'app déclenche un Flow – par exemple pour relever tous les stores immédiatement.

✅ Fin d'alerte

Lorsque l'alerte est levée, un Flow séparé est déclenché pour permettre de refermer les stores.

🔌 Surveillance de la connexion API

Si une requête API échoue, une alarme est déclenchée immédiatement et un Flow est activé.

⏱️ Surveillance (Watchdog)

Si aucune interrogation API réussie n'a eu lieu depuis plus de 10 minutes, un Flow est déclenché automatiquement.

Prérequis

  • Homey Pro avec accès internet
  • Inscription sur hagelschutz-einfach-automatisch.ch
  • ID de l'appareil (deviceId / numéro de série) et ID du type matériel (hwtypeId) valides

Capabilities / Capteurs

L'appareil «Capteur d'alerte grêle» fournit les valeurs suivantes sur la tuile de l'appareil :

🚨 Alerte grêle alarm_generic

Alarme booléenne – true quand currentState ≠ 0. Affichée par Homey comme indicateur d'alarme rouge.

# Signal grêle hail_state

Valeur numérique brute de l'API :

  • 0 – Pas de grêle
  • 1 – Alerte grêle active
  • 2 – Alerte grêle active (alarme de test)
🔌 Erreur de connexion API api_error_state

Alarme booléenne – true si la dernière requête API a échoué (pas de HTTP 200 ou délai dépassé).

🕐 Dernière interrogation API last_poll

Date et heure de la dernière interrogation API réussie au format JJ.MM, HH:MM.

Alarme de test

La connexion API peut être testée directement sur meteo.netitservices.com. Après connexion, activez le commutateur Alarme de test – dans les 120 secondes, le capteur devrait signaler currentState = 2 et déclencher le Flow. Désactivez ensuite l'alarme de test.

Vous pouvez également utiliser l'action Flow «Vérifier le statut grêle maintenant» pour déclencher une interrogation manuelle immédiate.

Questions fréquentes

Où obtenir l'ID de l'appareil et le hwtypeId ?

Après l'inscription sur hagelschutz-einfach-automatisch.ch vous recevez les identifiants par e-mail. L'ID de l'appareil est l'adresse MAC à 12 caractères de votre Signalbox ; le hwtypeId est un identifiant entier du type d'appareil.

Pourquoi l'intervalle d'interrogation est-il fixé à 120 secondes ?

La spécification API de l'AEAI impose un intervalle minimum de 120 secondes. Le système d'alerte grêle recalcule de toute façon les prévisions toutes les 5 minutes – des interrogations plus fréquentes n'apportent aucun avantage.

Que signifie currentState = 2 ?

currentState = 2 est une alarme de test pouvant être déclenchée manuellement via meteo.netitservices.com. L'app la traite de façon identique à une vraie alarme (non-zéro = grêle).

Le capteur affiche «non disponible» – que faire ?

  • Vérifier la connexion internet de Homey
  • Contrôler l'ID de l'appareil et le hwtypeId dans les paramètres
  • Se connecter à meteo.netitservices.com et vérifier le statut
  • Exécuter l'action Flow «Vérifier le statut grêle maintenant» et consulter les logs

L'app fonctionne-t-elle sans Signalbox ?

Oui – l'app utilise directement l'API METEO REST et ne nécessite pas de Signalbox physique (boîtier VKF). Elle la remplace pour les systèmes disposant de leur propre gestion technique du bâtiment.
